# gladys-tuya — Tuya external integration for Gladys Assistant

External integration that connects **Tuya / Smart Life** devices to
[Gladys Assistant](https://gladysassistant.com), built with the official
JavaScript SDK
[`@gladysassistant/integration-sdk`](https://github.com/GladysAssistant/integration-sdk-js).

It is a standalone port of the Gladys core Tuya service
(`server/services/tuya`) to the external-integration model: it runs in its own
Docker container next to Gladys, talks to the core through the SDK (outbound
WebSocket + REST), and its configuration UI is generated by Gladys from the
`config_schema` of the manifest.

## Supported devices

| Device type           | Features                                                                                                    | Modes         |
| --------------------- | ----------------------------------------------------------------------------------------------------------- | ------------- |
| Smart socket          | On/off control, child lock, power, voltage, current, energy metering (incl. LSC Power Plug FR)              | Cloud + local |
| Smart meter           | Power, voltage, current, forward/reverse energy (bidirectional meter)                                       | Cloud + local |
| Air conditioner       | On/off, mode (auto/cool/heat/dry/fan), target temperature, ambient temperature                              | Cloud + local |
| Pilot-wire thermostat | Fil pilote mode, child lock, setpoint, ambient temperature, window state, energy (RP5 family, Konyks eCosy) | Cloud + local |

## Configuration

The configuration form is generated by Gladys from the manifest. The fields
mirror the core Tuya service setup:

- **Endpoint**: Tuya data center region of your cloud project;
- **Client ID / Client Secret**: credentials of your
  [Tuya IoT platform](https://iot.tuya.com) cloud project;
- **App account UID**: UID of the Smart Life app account linked to the project;
- **Smart Life username** (optional): email or phone of the Smart Life account.

## Run it locally

```bash
npm install
GLADYS_HOST_API_URL="http://localhost:1443" \
GLADYS_INTEGRATION_TOKEN="<token>" \
GLADYS_INTEGRATION_SELECTOR="tuya" \
LOG_LEVEL=debug \
npm start
```

The three `GLADYS_*` variables are injected by the Gladys supervisor when the
integration runs inside its sandboxed container.
